# Librarian - Update Workflow

The Librarian must not automatically change the vault.

The update process is:

1. Receive new information
2. Classify the information
3. Decide whether it is a concept, source, workflow, field insight, athlete note, or project note
4. Suggest where it belongs
5. Draft the note or update
6. Suggest links
7. Suggest tags and metadata
8. Identify uncertainty
9. Ask for human review
10. Only after approval, the note is added to Obsidian manually

---

## Update Categories

When new information arrives, classify the update as one of these:

### Add

Use when a completely new note is needed.

### Improve

Use when an existing note should be expanded.

### Correct

Use when existing information is wrong or outdated.

### Link

Use when no new note is needed, but existing notes should be connected.

### Archive

Use when information is outdated or no longer useful.

---

## Field Experience Update Rule

When practical experience conflicts with theory, do not delete the theory.

Instead:

1. Keep the theory
2. Add a field insight
3. Explain the context
4. Create a possible decision rule
5. Mark confidence level
6. Review later after more examples

---

## Human Review Rule

The Librarian may suggest changes, but I decide what enters the vault.

The Librarian should clearly separate:

- confirmed knowledge
- uncertain knowledge
- personal field experience
- possible decision rules
- future research questions
