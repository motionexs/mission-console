# Librarian - Linking Rules

The Librarian must create useful links, not random links.

## Good Links

A link is useful when it answers one of these questions:

- What causes this?
- What does this affect?
- What does this help explain?
- Where is this used in coaching?
- What decision does this support?
- What workflow uses this concept?
- What evidence supports this idea?

## Bad Links

Avoid links that are too broad or weak.

Bad example:

Tendon stiffness links to sport.

Good example:

Tendon stiffness links to plyometric training because it affects elastic energy use and jump performance.

## Required Links by Note Type

Concept notes should link to:

- related concepts
- workflows that use the concept
- source notes that support the concept

Source notes should link to:

- concept notes created from the source
- workflows affected by the source

Workflow notes should link to:

- concepts used in the workflow
- field insights that improved the workflow
- source notes that support the workflow

Field insight notes should link to:

- concepts involved
- workflows that should change because of the insight
- athlete case notes if relevant

## Link Quality Rule

Every link should have a reason.

Do not link notes only because they are in the same general topic.

Good linking means the connection helps future coaching decisions.

## Field Insight Linking Rule

When suggesting links for Field Insight notes, separate links into:

### Existing links

Notes that already exist in the vault.

### Optional links

Existing notes that may be useful, but are not essential.

### Future notes to create

Useful notes that do not exist yet.
