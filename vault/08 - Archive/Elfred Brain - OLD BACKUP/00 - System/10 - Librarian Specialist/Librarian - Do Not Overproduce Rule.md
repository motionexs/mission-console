# Librarian - Do Not Overproduce Rule

## Purpose

This rule prevents the Librarian from creating too much at once.

The Performance Brain must grow slowly, clearly, and intentionally.

---

## Main Rule

The Librarian must not create too many notes, folders, tags, workflows, or links at once.

The goal is not to make the vault look full.

The goal is to make the vault useful.

---

## When Processing A Source

When I upload a PDF, lecture, book section, or research paper, the Librarian should:

1. Create one Source Note
2. Suggest possible Concept Notes
3. Recommend only one next Concept Note to create first
4. Wait before creating more

---

## When Processing Field Experience

When I give a coaching observation or practical example, the Librarian should:

1. Create one Field Insight Note
2. Link it to existing concepts
3. Link it to affected workflows
4. Suggest future notes only if needed
5. Avoid creating extra notes immediately

---

## When Creating Links

The Librarian should avoid random links.

A link must help explain:

- what caused something
- what it affects
- what decision it supports
- what workflow uses it
- what evidence supports it

---

## When Suggesting New Notes

The Librarian must separate new note suggestions into:

### Create now

The one most important note to create next.

### Create later

Useful notes that can wait.

### Maybe create later

Notes that may be useful but are not urgent.

---

## Quality Rule

A small number of strong notes is better than many weak notes.

The vault should grow through:

- real learning
- source processing
- coaching experience
- workflow improvement
- useful links

Not through random note creation.

---

## Final Rule

If unsure, the Librarian should ask:

What is the most useful note to create next?
